# Nameplates

Spaced capitals between paired marks. The pattern: `<mark> L E T T E R S <mark>`
— wide tracking turns a name into an emblem.

## Canon (current)

```
·  Z  I  N  Z  A  K  I  ·          quiet dot — default
—  Z  I  N  Z  A  K  I  —          dash — formal
⟦  Z  I  N  Z  A  K  I  ⟧          brackets — technical, HUD
◇  Z  I  N  Z  A  K  I  ◇          diamond — ceremonial
◸  Z  I  N  Z  A  K  I  ◿          corners — framed, poster-like
```

## Typeface variants (Unicode styles)

| Style | Sample | Use |
|---|---|---|
| mono | `𝚉 𝙸 𝙽 𝚉 𝙰 𝙺 𝙸` | terminals, tech contexts |
| bold sans | `𝗭 𝗜 𝗡 𝗭 𝗔 𝗞 𝗜` | chat display names |
| plain caps | `Z I N Z A K I` | maximum compatibility |

## Kanji suffix

A single kanji after the name adds a mood — never more than one:

```
◇  Z I N Z A K I  影  ◇        shadow
⟦  Z I N Z A K I  極  ⟧        extreme / utmost
·  Z I N Z A K I  玄  ·        profound darkness
芯 𝚣𝚒𝚗𝚣𝚊𝚔𝚒                     core (prefix form, lowercase intimate)
```

## Rules

1. Two spaces between letters; a hair-space or thin space also works where
   width is precious.
2. Marks always come in pairs (or a prefix kanji alone).
3. Unicode "fonts" (𝗯𝗼𝗹𝗱/𝚖𝚘𝚗𝚘) break screen readers and search — use them
   only in display contexts (profile names, banners), never in body text.
4. One nameplate per surface. It is a seal, not a watermark.

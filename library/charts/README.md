> Numbers made visible without leaving the terminal — and without heavy fills.
> A thin line with a gilded head, dots, or braille. Gold is a line here too,
> not a block. Color stays in role; warmth appears only at a threshold.

```
sparkline.txt   a trend in one line — braille dots or low blocks (no █)
bars.txt        compare values — thin track + gilded head (or ▰ squares)
gauge.txt       one value vs its whole — line with a head, dotted rest
```

## Principles
1. **No slabs.** Never a run of `█` or a `[████]` box — it reads heavy.
   A line with a head (`───●┄┄`), dots, or braille carries the value lighter.
2. **Gold is a line, not a fill** — the same gilding rule as everywhere.
3. **Shape first, number second**; label left and dim, value right and bone.
4. **Warmth only at a threshold** — persimmon past warn, scarlet past critical;
   otherwise the chart stays gold/bone (rarity = power).
5. **Braille = 2×4 resolution** for density; low blocks (`▁▂▃▄`) for a lighter
   touch — peaks capped, never the full block.
